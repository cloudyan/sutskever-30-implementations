# 论文 25：Kolmogorov 复杂度 — 什么是信息？

> **核心问题**：一段数据到底包含多少"信息"？什么是"随机"？什么是"规律"？

**Notebook**: `25_kolmogorov_complexity.ipynb`

---

## 一、核心定义

### 1.1 一句话定义

> **K(x) = 能输出 x 的最短程序的长度**

这就是"信息"的精确数学定义。

### 1.2 直观理解

```
数据A: 000000000000000000000000000000  （30个零）
  → 程序: print('0' * 30)  → 约 20 字符
  → K(A) ≈ 20  → 信息量很少

数据B: 101100101110010111001011100101  （看起来随机）
  → 程序: print("101100101110010111001011100101")  → 约 40 字符
  → K(B) ≈ 40  → 信息量很大（几乎等于数据本身长度）
```

**信息 = 压缩的难度。越有规律，信息量越少。**

---

## 二、数学基础

### 2.1 形式化定义

```
K(x) = min |p|
       p: U(p) = x

其中：
  U = 通用图灵机（通用解释器）
  p = 程序
  |p| = 程序长度（比特数）
```

### 2.2 不变性定理

**问题**：K(x) 依赖具体的编程语言，那不就不客观了？

**定理**：对任意两台通用机 U₁ 和 U₂，存在常数 c，使得：

```
|K_U1(x) - K_U2(x)| ≤ c
```

**直觉**：换语言只会多一个"翻译器"的固定开销。对足够长的 x，差异是常数项。

### 2.3 与 Shannon 熵的关系

| 概念 | 定义 | 特点 |
|------|------|------|
| **Shannon 熵** | H(X) = -Σp(x)log₂p(x) | 需要概率分布，衡量随机变量的**平均**信息量 |
| **Kolmogorov 复杂度** | K(x) = 最短程序长度 | 不需要分布，衡量**单个样本**的信息量 |

**理论关系**：对典型样本，E[K(X)] ≈ H(X) × |x| + O(log|x|)

## 三、三个核心洞察

### 3.1 随机 = 不可压缩

```
如果一个字符串无法被任何程序缩短输出，它就是"随机"的

这给了"随机"一个精确的数学定义！
不再是"看起来乱"这种模糊说法
```

### 3.2 大多数字符串都是随机的

**证明**：

```
长度为 n 的二进制串有 2^n 个
长度小于 n 的程序只有 2^(n-1) + 2^(n-2) + ... < 2^n 个

结论：至少一半的字符串是不可压缩的（随机的）
```

### 3.3 K(x) 是不可计算的！

**Berry 悖论**：

```
假设存在算法 ComputeK(x)，能算出任意字符串的 K(x)

那么我们可以写程序 P：
  "打印第一个满足 K(x) > 1000 的字符串 x"

程序 P 本身才约 100 字符
但它却生成了一个 K(x) > 1000 的字符串
→ 矛盾！

结论：K(x) 不可计算
```

**本质原因**：计算 K(x) 需要解决停机问题。

## 四、Solomonoff 归纳：终极预测机器

### 4.1 核心思想

> 如何预测下一个符号？把所有能生成已观测数据的程序按长度加权（越短权重越大），然后用它们各自预测下一个符号，加权平均。

### 4.2 分步拆解

**第 1 步**：找到所有"解释"数据的程序

```
假设已观测到数据：2, 4, 6, 8, 10, ?

找出所有能输出 2, 4, 6, 8, 10 的程序：
  程序 A：print([2, 4, 6, 8, 10])  → 硬编码
  程序 B：for i in range(1, 6): print(2*i)  → 偶数序列
  程序 C：...其他复杂程序...
```

**第 2 步**：按程序长度加权

```
权重公式：weight(P) = 2^(-|P|)

程序 A（长度 20）：权重 = 2^-20
程序 B（长度 35）：权重 = 2^-35
程序 C（长度 100）：权重 ≈ 0

关键：短程序获得指数级更高的权重！
```

**第 3 步**：每个程序预测下一个符号

```
程序 A 的预测：无（只是硬编码，没有规律）
程序 B 的预测：12（下一个偶数）
程序 C 的预测：...
```

**第 4 步**：加权投票

```
P(下一个=12) = Σ weight(P_i) × [P_i 预测 12]

选择概率最高的预测 → 12
```

### 4.3 为什么这是"最优"的？

**数学定理**：Solomonoff 归纳的预测误差随数据量增加而指数级下降，**没有任何其他算法能比它更快**。

这是理论上完美的学习算法 — 虽然不可计算，但定义了"学习"的理想极限。

## 五、与 AI 的关系

### 5.1 核心对应

| AI 概念 | Kolmogorov 解释 |
|---------|-----------------|
| 奥卡姆剃刀（简单即美） | 简单模型 = 短程序 = 低 K(x) |
| 学习 = 发现压缩 | 学会乘法口诀 = 压缩了所有乘法事实 |
| 过拟合 = 死记硬背 | K(模型) 大但 K(数据\|模型) 小 = 只是记住噪声 |
| 泛化 = 真正理解 | K(模型) + K(数据\|模型) 小 = 发现了真正的规律 |

### 5.2 算法概率与 Occam's Razor

```
Solomonoff 的洞察：P(x) ≈ 2^(-K(x))

意思是：
  - 简单的序列（K 小）有更高先验概率
  - 复杂的序列（K 大）先验更低

这就是 Occam's Razor 的数学化！
```

### 5.3 为什么正则化/剪枝/MDL 有效？

```
一旦接受"学习 = 压缩"，很多工程现象就合理了：

正则化（L1/L2）：
  → 惩罚复杂度，偏向低 K 的模型

剪枝/量化/蒸馏：
  → 让模型更可压缩，也常常泛化更好

MDL（论文 23）：
  → 在可计算范围内近似 K(x)
```

## 六、实际估计 K(x)

### 6.1 用压缩算法做上界

K(x) 不可计算，但可以用压缩算法做近似：

```python
import zlib

def estimate_kolmogorov(s):
    """用 zlib 压缩估计 K(x) 的上界"""
    if isinstance(s, str):
        s = s.encode('utf-8')
    return len(zlib.compress(s, level=9))

# 测试不同类型的字符串
strings = {
    "全零": "0" * 1000,
    "重复模式": "ABC" * 333,
    "英文文本": "the quick brown fox " * 50,
    "随机二进制": ''.join([str(random.randint(0,1)) for _ in range(1000)])
}

for name, s in strings.items():
    original = len(s)
    compressed = estimate_kolmogorov(s)
    ratio = compressed / original
    print(f"{name}: {original} → {compressed} 字节 (压缩率: {ratio:.3f})")
```

### 6.2 压缩率解读

```
压缩率 < 0.1：高度结构化（K(x) 很小）
压缩率 ≈ 1.0：随机（K(x) ≈ |x|）
压缩率 > 1.0：压缩开销（非常短的字符串）
```

## 七、一句话总结

> **K(x) 衡量的是：你至少需要多少信息才能描述 x。**
> 
> **压缩 = 理解。学习 = 找到更短的描述。智能 = 高效的压缩。**
