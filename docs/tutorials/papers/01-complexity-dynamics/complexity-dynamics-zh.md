## 引言：一个关于时间的深刻问题

几周前，我有幸参加了 FQXi 举办的 "Setting Time Aright"（校准时间）会议，部分议程是在从挪威卑尔根到丹麦哥本哈根的游轮上进行的。（为什么理论计算机科学会议从不在游轮上举办呢？至少这样可以减少参会者偷偷溜出会场的情况。）这次会议汇聚了物理学家、宇宙学家、哲学家、生物学家、心理学家，以及（出于某种奇怪的原因）一位量子复杂性博主，共同探讨时间的存在性、方向性和本质。

Sean Carroll 在会议的[开幕演讲](http://www.slideshare.net/seanmcarroll/setting-time-aright)中提出了一个美妙的问题：

> **为什么物理系统的"复杂性"或"有趣性"似乎随时间先增加后达到最大值再减少，而熵却单调递增？**

本文的目的是概述一个可能的答案，该答案借鉴了**柯尔莫哥洛夫复杂性（Kolmogorov Complexity）**的概念。如果这个答案之前已经被提出过，我相信会有人告诉我。

---

## 背景：熵与第二定律

首先，一些背景知识：我们都知道[热力学第二定律](http://en.wikipedia.org/wiki/Second_law_of_thermodynamics)，它指出任何封闭系统的**熵**会随时间增加，直到达到最大值。这里的"熵"很难精确定义——我们稍后会回到这个问题——但它以某种方式衡量了一个系统的"随机性"、"通用性"或"无序程度"。

正如 Sean 在他精彩的书《From Eternity to Here》中指出的，第二定律**几乎**是一个同义反复：一个系统怎么可能不倾向于演化到更"通用"的配置呢？如果它不这样做，那些配置就不会是"通用"的！

所以真正的问题不是为什么熵在增加，而是为什么它一开始如此低。换句话说，为什么宇宙在大爆炸时的初始状态包含如此多的秩序，供宇宙随后的演化来破坏？本文不会探讨那个著名的谜团，而只是将初始状态的低熵视为给定的。

---

## 核心观察：复杂性 ≠ 熵

让我们感兴趣的一点是：尽管孤立物理系统的熵单调增加，但它们**并非**单调地变得更加"复杂"或"有趣"。

Sean 在这里没有定义他所说的"复杂"或"有趣"是什么意思——事实上，定义这些概念本身就是他的挑战的一部分——但他用一个咖啡杯的例子来说明他的想法。

![咖啡杯混合过程](https://scottaaronson.blog/coffee-small.jpg)

从左到右，熵单调增加，但直观上，"复杂性"似乎在**中间**那张图片中最高：那张带有所有牛奶丝状结构的图片。整个宇宙也是如此：大爆炸后不久，宇宙基本上只是低熵的高能粒子汤。一古戈尔年（10^100年）后，在最后一批黑洞以霍金辐射的形式消散后，宇宙基本上将只是**高熵**的**低能**粒子汤。但在今天，在中间阶段，宇宙包含了有趣的结构，如星系、大脑和热狗形状的新奇车辆。

## 挑战的双重性

在回答 Sean 的挑衅性问题（是否存在某种"复杂动力学定律"可以解释他的图表）时，我认为挑战是双重的：

1. **提出一个合理的"复杂性"的形式化定义**
2. **证明如此定义的"复杂性"在自然模型系统中在中间时间较大，尽管在初始时间和晚期时间接近零**

澄清一下：解释为什么复杂性在初始时间应该接近零并不难。这是因为我们假设**熵**接近零，而熵可能给出了复杂性的上限。解释为什么复杂性在晚期时间应该接近零也不难：这是因为系统达到了平衡态（即类似于所有可能状态的均匀分布），我们本质上**定义**它为简单的。

在中间时间，这两个约束都不起作用，因此复杂性**可能**变得很大。但它**确实**变得很大吗？有多大？我们如何预测？我们谈论的是什么样的"复杂性"？

---

## 柯尔莫哥洛夫复杂性：一个可能的答案

在断断续续地思考这些问题后，我现在猜想它们可以用**精巧度（Sophistication）**的概念来回答，这是从[柯尔莫哥洛夫复杂性](http://en.wikipedia.org/wiki/Kolmogorov_complexity)理论中来的一个概念。

回顾一下，字符串 x 的**柯尔莫哥洛夫复杂性**是输出 x 的最短计算机程序的长度（在某个图灵通用编程语言中——确切的选择可以证明并不重要）。精巧度是一个更……嗯，精巧的概念，但我们稍后会讲到。

### 用柯尔莫哥洛夫复杂性定义熵

作为第一步，让我们用柯尔莫哥洛夫复杂性来定义**熵**。如何做到这一点已经不太明显了。如果你启动一个元胞自动机，或一个台球系统，从某个简单的初始配置开始，然后根据动力学定律让它演化一段时间，视觉上看起来熵在上升。但如果系统恰好是**确定性的**，那么在数学上，它的状态总是可以通过给出 (1) 初始状态，和 (2) 它已经运行的时间步数 t 来指定。前者需要常数比特数来指定（与 t 无关），而后者需要 log(t) 比特。因此，如果我们用柯尔莫哥洛夫复杂性作为熵的替代，那么熵最多只能随 t **对数增长**——比我们直观期望的线性或多项式增长慢得多。

解决这个问题至少有两种方法：

1. **考虑概率系统**，而不是确定性系统。在概率情况下，柯尔莫哥洛夫复杂性确实以多项式速率增长，正如你所期望的。

2. **用资源受限的柯尔莫哥洛夫复杂性**替代柯尔莫哥洛夫复杂性：在最短时间内输出状态的最短计算机程序的长度（或输出状态的最小深度-3电路的大小——就目前目的而言，施加什么样的资源限制并不重要，只要限制足够严格）。

即使有一个只有 log(t) 比特长的计算机程序来计算系统在 t 时间步后的状态，那个程序通常会使用随 t 增长（甚至更快）的**时间**量，所以如果我们排除足够复杂的程序，我们可以再次让我们的程序大小以多项式速率随 t 增长。

## 精巧度（Sophistication）：定义"复杂熵"

那么 Sean 所说的"复杂性"呢——为了避免与其他类型的复杂性混淆，从现在起我将称之为**"复杂熵（complextropy）"**？

为此，我们需要一组相关的概念，它们以精巧度、柯尔莫哥洛夫结构函数和算法统计学的名义出现。

背景故事是，在 1970 年代（引入柯尔莫哥洛夫复杂性**之后**），柯尔莫哥洛夫做了一个与 Sean 上述观察密切相关的观察。他说，一个均匀随机的字符串具有接近最大的柯尔莫哥洛夫复杂性，但它也是**最不**"复杂"或"有趣"的字符串之一。毕竟，我们可以通过说"它是随机的"来描述关于这个字符串你几乎想知道的一切！但是否有办法形式化这种直觉呢？确实有。

### 精巧度的形式化定义

首先，给定一个 n 位字符串的集合 S，让 K(S) 是输出 S 的元素然后停止的最短计算机程序中的比特数。此外，给定这样一个集合 S 和 S 的一个元素 x，让 K(x|S) 是给定一个用于测试 S 中成员资格的预言机时，输出 x 的最短程序的长度。

然后我们可以让 x 的**精巧度**，或 Soph(x)，是所有满足以下条件的集合 S 的 K(S) 的最小可能值：

1. x ∈ S
2. K(x|S) ≥ log₂(|S|) – c，对于某个常数 c

（换句话说，你可以通过说 x 属于 S 来提炼 x 中所有的"非随机"信息。）

直观上，Soph(x) 是最短计算机程序的长度，该程序描述的不一定是 x 本身，而是一个集合 S，x 是 S 的"随机"或"通用"成员。

为了说明，任何具有小柯尔莫哥洛夫复杂性的字符串 x 都具有小精巧度，因为我们可以让 S 是单元素集合 {x}。然而，一个均匀随机字符串**也**具有小精巧度，因为我们可以让 S 是所有 n 位字符串的集合 {0,1}ⁿ。

事实上，问题出现了：**是否存在任何精巧的字符串**！显然，在柯尔莫哥洛夫于 1980 年代初提出这个问题后，Alexander Shen 给出了肯定的回答（更多信息，请参阅 Gács、Tromp 和 Vitányi 的[这篇论文](http://homepages.cwi.nl/~paulv/papers/algorithmicstatistics.pdf)）。构造是通过对角线论证完成的，太复杂了，无法在这篇博客文章中详述。

## 为什么精巧度还不够

但这与咖啡杯有什么关系呢？

乍一看，精巧度似乎恰好具有我们在"复杂熵"度量中寻找的属性：它对于简单字符串**和**均匀随机字符串都很小，但对于"既不简单也不随机"的奇怪第三类字符串很大。

不幸的是，正如我们上面定义的，精巧度仍然无法完成这项工作。

对于**确定性系统**，问题与之前指出的柯尔莫哥洛夫复杂性的问题相同：我们总是可以通过指定初始状态、转移规则和 t 来描述系统在 t 时间步后的状态。因此，精巧度永远不会超过 log(t)+c。

即使对于**概率系统**，我们可以通过指定初始状态、概率转移规则和 t 来指定在 t 时间步后**所有可能状态的集合 S(t)**。而且，至少假设 S(t) 上的概率分布是均匀的，通过简单的计数论证，t 步后的状态几乎总是 S(t) 的"通用"元素。所以同样，精巧度几乎永远不会超过 log(t)+c。

---

## 解决方案：引入计算资源限制

如何解决这个问题？我认为关键是将**计算资源限制**引入图景。（我们在熵的讨论中已经看到了这一点。）

特别是，假设我们将 n 位字符串 x 的复杂熵定义为类似于以下内容：

> **在 n log(n) 时间内运行的最短计算机程序中的比特数，该程序输出一个集合 S 的近乎均匀样本，使得 (i) x∈S，且 (ii) 任何在 n log(n) 时间内输出 x 的计算机程序，给定一个提供来自 S 的独立、均匀样本的预言机，至少有 log₂(|S|)-c 比特，对于某个常数 c。**

这里 n log(n) 只是作为复杂性限制的具体例子：可以用其他时间限制替换，或限制为（比如说）常数深度电路或其他弱计算模型。

这个定义的动机是，我们想要**某种**"复杂熵"度量，它将给图片中的第一和第三个咖啡杯分配接近 0 的值，但给第二个咖啡杯分配一个大值。因此，我们考虑最短高效计算机程序的长度，该程序输出的不一定是目标字符串 x 本身，而是来自概率分布 D 的样本，使得 x 相对于 D 不是高效可压缩的。（换句话说，对任何高效算法来说，x 看起来像是来自 D 的"随机"或"通用"样本。）

### 双重效率约束的重要性

请注意，对于这个定义，我们在**两个**地方施加了计算效率要求是至关重要的：在采样算法上，**也**在重建 x 的算法上（给定采样预言机）。

- 没有**第一个**效率约束，复杂熵根据前面的论证永远不会超过 log(t)+c。
- 没有**第二个**效率约束，复杂熵**会**增加，但它可能会继续增加，原因如下：时间受限的采样算法无法从**确切**正确的集合 S 采样，只能采样一个合理的近似，而一个具有**无限时间**的重建算法可能能够使用目标字符串 x 的特殊属性来用少于 log₂(|S|)-c 的比特重建 x。

但只要我们记得在**两个**算法上都施加计算效率要求，我**猜想**复杂熵将满足"复杂动力学第一定律"，表现出 Sean 想要的精确行为：初始状态小，中间状态大，然后在混合完成后再次变小。

## 开放问题与未来方向

我还不知道如何证明这个猜想。但关键的是，它**不是**一个为了展示自己思想有多广泛而抛出的无望的开放式问题，而是一个相对有界的问题，关于它实际定理可以被证明，实际论文可以被发表。

如果你想这样做，第一步将是把我上面说的一切都"实例化"到特定的模型系统和特定的资源限制上。一个好的选择可能是一个离散化的"咖啡杯"，由黑白像素（"咖啡"和"牛奶"）的 2D 数组组成，最初处于分离的组件中，然后受到随机最近邻混合动力学的影响。（例如，在每个时间步，我们均匀地随机选择一个相邻的咖啡像素和牛奶像素，并交换两者。）

我们能否证明对于这样的系统，复杂熵在中间时间变得很大（直观上，因为需要指定全黑像素区域、全白像素区域和混合黑白像素区域之间的不规则**边界**）？

人们可以尝试从理论上或经验上证明这样的陈述。

- **理论上**：我完全不知道如何证明它，尽管有一种清晰的直觉认为这样的陈述应该成立：让我把它作为一个精彩的（我认为）开放问题抛出来！
- **经验上**：人们可以简单地尝试在模拟系统中**绘制**复杂熵，比如离散咖啡杯，并展示它具有预测的小-大-小行为。

这里的一个明显困难是，根据我给出的任何定义，复杂熵几乎肯定是难以计算甚至近似的。然而，人们可以尝试像许多其他人一样，在受柯尔莫哥洛夫复杂性启发的实证研究中解决这个问题：即，通过使用你**可以**计算的东西（例如，gzip 压缩文件的大小）作为你**不能**计算的东西（例如，柯尔莫哥洛夫复杂性 K(x)）的粗略替代。

为了完全披露，一位优秀的 MIT 本科生 Lauren Oullette 最近和我一起开始了一个研究项目，她正在尝试做完全相同的事情。所以希望，到学期末，我们将能够至少在物理学的严格程度上回答 Sean 的问题！在数学/计算机科学的严格程度上回答这个问题可能需要更长的时间。
