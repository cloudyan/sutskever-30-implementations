# 03 香农熵 vs 算法熵：两种度量信息的方式

问下大家，如果要你量化"信息"，你会怎么做？

Claude Shannon 说：看概率分布！而 Andrey Kolmogorov 说：看最短程序！这两种方法看起来完全不同，但它们之间有什么联系呢？

今天，我们就来深入理解这两种信息度量方式的关系！

## 香农熵：基于概率的信息度量

### 定义回顾

**香农熵**（1948）度量随机变量的不确定性：

$$H(X) = -\sum_{i} p(x_i) \log_2 p(x_i)$$

**直观理解**：
- 确定事件（概率=1）：熵 = 0（没有信息）
- 均匀分布：熵最大（最难预测）
- 有偏分布：熵中等（有一定可预测性）

### 关键性质

1. **非负性**：$H(X) \geq 0$
2. **最大性**：均匀分布时达到最大值 $H(X) \leq \log_2 n$
3. **可加性**：独立变量的联合熵等于熵之和 $H(X,Y) = H(X) + H(Y)$
4. **条件减少**：$H(X|Y) \leq H(X)$（知道更多信息不会增加不确定性）

### 示例

**公平硬币**：
$$H = -0.5 \log_2 0.5 - 0.5 \log_2 0.5 = 1 \text{ bit}$$

**有偏硬币**（正面概率 0.9）：
$$H = -0.9 \log_2 0.9 - 0.1 \log_2 0.1 \approx 0.47 \text{ bits}$$

**确定性硬币**（总是正面）：
$$H = 0 \text{ bits}$$

## 算法熵（Kolmogorov 复杂度）：基于描述的信息度量

### 定义回顾

**Kolmogorov 复杂度**（1965）度量描述对象所需的最短程序长度：

$$K(x) = \min_{p} \{|p| : U(p) = x\}$$

**直观理解**：
- 简单对象：KC 低（短程序即可生成）
- 复杂对象：KC 高（需要长程序）
- 随机对象：KC ≈ 对象长度（无法压缩）

### 关键性质

1. **不可计算性**：不存在通用算法计算 $K(x)$
2. **通用性**：与选择的编程语言无关（差一个常数）
3. **不可压缩性**：大多数字符串的 $K(x) \approx |x|$
4. **单调性**：$K(x) \leq |x| + O(1)$

### 示例

**字符串 "000...0"**（10000个0）：
```python
print("0" * 10000)  # 程序长度 ≈ log₂(10000) ≈ 14 比特
```
$K(x) \approx 14$ 比特

**字符串 "ababab..."**（5000个"ab"）：
```python
print("ab" * 5000)  # 程序长度 ≈ log₂(5000) ≈ 13 比特
```
$K(x) \approx 13$ 比特

**随机字符串**（无规律）：
```python
print("xJ#9mK$pL2vN...")  # 只能硬编码
```
$K(x) \approx n$（字符串长度）

## 两种熵的对比

| 特性 | 香农熵 | 算法熵 (KC) |
|------|--------|-------------|
| **提出者** | Claude Shannon (1948) | Andrey Kolmogorov (1965) |
| **基础** | 概率论 | 计算理论 |
| **度量对象** | 随机变量 | 单个对象 |
| **取值** | 期望值 | 确定值 |
| **计算难度** | 可计算（知道分布） | 不可计算 |
| **依赖先验** | 需要概率分布 | 不需要 |
| **适用范围** | 统计系统 | 个体对象 |

## 深刻的联系：平均情况 vs 最坏情况

虽然两种熵看起来不同，但它们之间有深刻的数学联系！

### 香农熵是 KC 的期望值

**定理**：如果从分布 $D$ 中随机抽取字符串 $x$，则：

$$\mathbb{E}_{x \sim D}[K(x)] \approx H(D)$$

换句话说：**香农熵是 Kolmogorov 复杂度的期望值！**

### 证明思路

1. **上界**：$K(x) \leq -\log_2 D(x) + O(1)$
   - 可以用 Shannon-Fano-Elias 编码构造程序
   - 概率越大的字符串，编码越短

2. **下界**：对于随机抽取的 $x$，$K(x) \geq -\log_2 D(x) - O(1)$
   - 由 KC 的计数论证
   - 短程序只能描述少数字符串

3. **取期望**：
   $$\mathbb{E}[K(x)] = \sum_x D(x) K(x) \approx \sum_x D(x) (-\log_2 D(x)) = H(D)$$

### 直观理解

- **香农熵**：度量"平均需要多少比特来描述"
- **算法熵**：度量"具体需要多少比特来描述这个特定对象"

对于随机抽取的对象，两者大致相等。但对于特定对象，KC 可能远小于香农熵（如果对象有结构）。

## 算法随机性：统一两种视角

### 马丁-洛夫随机性

**定义**（Per Martin-Löf, 1966）：

字符串 $x$ 是**算法随机的**，如果它通过所有有效的统计测试。

等价地：$x$ 是随机的，如果 $K(x) \geq |x| - c$（对于某个常数 $c$）。

### 与香农熵的关系

**定理**：长度为 $n$ 的随机字符串的 KC 约等于 $n$。

这意味着：
- **香农视角**：随机字符串的熵最大（均匀分布）
- **算法视角**：随机字符串的 KC 最大（无法压缩）

两者在"随机性"这个概念上达成了一致！

## 实际应用中的选择

### 什么时候用香农熵？

✅ **适合场景**：
- 已知概率分布
- 关心平均行为
- 统计物理、通信理论
- 机器学习中的交叉熵损失

**示例**：
- 数据压缩（Huffman 编码）
- 信道容量计算
- 决策树的信息增益

### 什么时候用算法熵？

✅ **适合场景**：
- 不知道概率分布
- 关心特定对象
- 算法信息论、计算复杂性
- 最小描述长度（MDL）学习

**示例**：
- 评估模型的复杂度
- 检测数据的规律性
- 理解随机性的本质

## 统一框架：算法统计

### 算法充分统计量

给定数据 $x$，我们想知道：
- 哪些信息是"结构"（可压缩的）？
- 哪些信息是"噪声"（随机的）？

**算法充分统计量**：一个最短的程序，它描述了 $x$ 的"结构"，而将"随机"部分留给无条件描述。

形式化：
$$K(x) = K(S) + \log_2 |S| + O(1)$$

其中 $S$ 是包含 $x$ 的集合，$K(S)$ 描述结构，$\log_2 |S|$ 描述随机部分。

### 与香农统计的类比

| 香农统计 | 算法统计 |
|---------|---------|
| 充分统计量 $T(X)$ | 算法充分统计量 $S$ |
| 保留所有关于参数的信息 | 保留所有关于"结构"的信息 |
| 数据压缩：$H(X) = H(T) + H(X\|T)$ | KC 分解：$K(x) = K(S) + K(x\|S)$ |

## 小结

问下大家，现在理解两种熵的关系了吗？

**核心要点**：

1. **香农熵**：基于概率，度量平均不确定性，可计算
2. **算法熵**：基于描述，度量特定对象的复杂度，不可计算
3. **深刻联系**：$\mathbb{E}[K(x)] \approx H(D)$（期望值相等）
4. **统一视角**：算法随机性统一了两种对"随机"的理解

**关键洞察**：
- 香农熵是"平均视角"，算法熵是"个体视角"
- 对于随机对象，两者一致；对于结构化对象，算法熵更小
- 算法统计提供了统一框架，区分"结构"和"随机"

**为什么这很重要？**

理解这两种熵的关系，让我们能够：
1. 在不同场景选择合适的信息度量
2. 理解数据压缩的理论极限
3. 评估机器学习模型的复杂度
4. 探索"信息"的本质
