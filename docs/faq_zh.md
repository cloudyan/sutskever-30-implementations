## 🧠 概念理解与对比

### Q5：RNN、LSTM、Transformer有什么区别？
**一句话结论**：**记忆方式不同**，适用于不同场景。

**对比表**：
| 技术 | 记忆方式 | 优点 | 缺点 | 适用场景 |
|------|----------|------|------|----------|
| **RNN** | 简单隐藏状态 | 简单、计算快 | 记不住长序列 | 短文本、简单序列 |
| **LSTM** | 三个门控制 | 能记长序列 | 计算稍复杂 | 语音识别、机器翻译 |
| **Transformer** | 注意力看全句 | 并行、效果好 | 内存消耗大 | 大语言模型、长文本 |

**选择指南**：
- **短序列**（<100词）→ RNN或LSTM
- **长序列**（>100词）→ Transformer
- **需要并行训练** → Transformer
- **资源有限** → RNN或LSTM

### Q6：CNN为什么比传统方法好？
**一句话结论**：**自动学习特征**，而不是手工设计特征。

**传统方法**：
```
手工设计特征 → 分类器
   ↓
边缘检测 + 颜色直方图 + 纹理特征 → SVM分类
```

**CNN方法**：
```
原始图片 → 卷积层自动学特征 → 分类器
```

**3步理解法**：
1. **第1层**：学边缘检测器
2. **第2层**：学纹理检测器
3. **第3层**：学物体部件检测器

**实际优势**：
- ✅ **准确率更高**：ImageNet比赛从70%提升到90%+
- ✅ **开发更快**：不用手工设计特征
- ✅ **泛化更好**：在新数据上表现更好

### Q7：注意力机制是什么？
**一句话结论**：**动态聚焦**，看输入的不同部分。

**类比**：
- **传统方法**：把整本书压缩成一句话摘要（信息丢失）
- **注意力方法**：翻译时随时回头翻书（信息完整）

**核心机制**：
```
输入： [苹果, 红了, 因为, 昨天, 刮了风]
生成"红"时：关注"苹果"
生成"因为"时：关注"红了"
生成"昨天"时：关注"刮了风"
```

**3种注意力**：
1. **Bahdanau注意力**（论文14）：原始注意力，用于机器翻译
2. **自注意力**（论文13）：Transformer的核心，每个词看所有词
3. **多头注意力**：多个"视角"同时看，更全面

### Q8：残差连接为什么有效？
**一句话结论**：**修了信息高速公路**，让梯度能直接传回去。

**问题**：网络太深时，梯度"消失"（像水通过多层滤网）

**解决方案**：
```
传统网络：y = F(x)  # 学习完整答案
残差网络：y = F(x) + x  # 只学习修正值
```

**直观理解**：
- 如果F(x)学不会 → y = 0 + x = x（至少能传回去）
- 如果F(x)学会了 → y = 修正值 + x（学得更好）

**实际效果**：
- 可以训练1000+层网络
- ImageNet比赛冠军
- 现代所有深度网络的基础

## 🧮 数学恐惧与公式理解

### Q9：看到公式就害怕怎么办？
**一句话结论**：**把公式当"接口说明书"**，不是考试题。

**公式恐惧症解药**：
1. **第一步**：只看符号含义（不看推导）
2. **第二步**：找代码对应关系
3. **第三步**：用最小数值例子验证

**示例：LSTM公式**：
```
f_t = σ(W_f · [h_{t-1}, x_t] + b_f)  # 忘记门
i_t = σ(W_i · [h_{t-1}, x_t] + b_i)  # 输入门
```

**翻译**：
- `σ` = sigmoid函数（0到1）
- `W_f` = 忘记门的权重矩阵
- `[h_{t-1}, x_t]` = 拼接上一时刻隐藏状态和当前输入
- `b_f` = 忘记门的偏置

**代码对应**：
```python
def lstm_cell(x, h_prev):
    # 拼接
    concat = np.concatenate([h_prev, x], axis=-1)
    
    # 忘记门
    f = sigmoid(np.dot(W_f, concat) + b_f)
    
    # ... 其他门类似
    return f, i, o, c_new, h_new
```

### Q10：Q/K/V到底是什么意思？
**一句话结论**：**查询/键/值**，像图书馆检索系统。

**图书馆类比**：
- **Q（查询）**：你想找什么书？（用户的问题）
- **K（键）**：书的索引号（文档的标识）
- **V（值）**：书的内容（文档的实际内容）

**数学公式**：
```
Attention(Q,K,V) = softmax(Q·K^T/√d_k) · V
```

**形状说明**：
```
Q: (batch, seq_len_q, d_model)   # 查询
K: (batch, seq_len_k, d_model)   # 键
V: (batch, seq_len_v, d_model)   # 值
```

**3步理解法**：
1. **Q·K^T**：计算查询和键的相似度（相关性分数）
2. **softmax**：归一化，得到注意力权重（哪个键最相关）
3. **· V**：用权重加权值向量（得到输出）

### Q11：KL散度在VAE中惩罚什么？
**一句话结论**：**惩罚潜在空间偏离标准正态分布**。

**直观理解**：
- **编码器输出**：均值μ和方差σ（描述潜在变量的分布）
- **我们希望**：潜在变量服从标准正态分布N(0,1)
- **KL散度**：衡量编码器分布和标准正态分布的差异

**公式翻译**：
```
KL(q(z|x) || p(z)) = -0.5 * Σ(1 + log(σ²) - μ² - σ²)
```

**各项含义**：
- `log(σ²)`：鼓励方差不要太小（避免过拟合）
- `μ²`：鼓励均值接近0（中心化）
- `σ²`：鼓励方差接近1（标准化）

**实际效果**：
- 防止编码器"作弊"（比如把所有样本映射到同一个点）
- 让潜在空间连续可插值（可以生成新样本）

### Q12：CTC为什么不用对齐？
**一句话结论**：**自动学习对齐**，不需要人工标注对齐信息。

**传统语音识别问题**：
```
音频: [s, i, l, e, n, t]  # 6个音素
文字: "silent"            # 6个字母
对齐: s→s, i→i, l→l, e→e, n→n, t→t  # 需要人工标注！
```

**CTC解决方案**：
1. **允许重复和空白**：s s i i l l e e n n t t
2. **自动合并重复**：s i l e n t
3. **删除空白**：得到最终结果

**数学原理**：
- **空白符号**（`_`）：表示没有输出
- **动态规划**：计算所有可能对齐的概率
- **损失函数**：最大化正确对齐的概率

**优势**：
- ✅ 不需要对齐标注
- ✅ 处理变长序列
- ✅ 端到端训练

### Q14：代码出错了怎么办？
**一句话结论**：**90%的错误是形状不匹配**，检查张量形状！

**常见错误排查清单**：

#### 错误1：形状不匹配（Shape mismatch）
**症状**：`ValueError: shapes (32,10) and (20,5) not aligned`
**解决方案**：
1. 打印每个张量的形状 `print(x.shape)`
2. 检查矩阵乘法规则 `(m,n) @ (n,p) → (m,p)`
3. 检查是否需要转置 `.T`

#### 错误2：NaN/Inf出现
**症状**：`loss`变成`nan`或`inf`
**解决方案**：
1. 检查学习率是否太大（尝试减小10倍）
2. 检查梯度是否爆炸（`print(grad)`看数值范围）
3. 添加梯度裁剪 `grad = np.clip(grad, -1, 1)`

#### 错误3：loss不下降
**症状**：`loss`一直不变或波动很大
**解决方案**：
1. 检查学习率（太小不下降，太大震荡）
2. 检查数据是否正确加载
3. 检查模型是否过小（增加层数或神经元）
4. 检查激活函数是否合理（ReLU比sigmoid好）

#### 错误4：内存不足
**症状**：`MemoryError`或`Killed`
**解决方案**：
1. 减小`batch_size`（从32减到16或8）
2. 减少序列长度
3. 使用更小的模型

### Q15：怎么修改代码做实验？
**一句话结论**：**从安全的修改开始**，逐步验证效果。

**3个安全实验**（不会破坏系统）：

#### 实验1：修改超参数
```python
# 原代码
learning_rate = 0.001
batch_size = 32
hidden_size = 128

# 修改为
learning_rate = 0.0005  # 减小学习率
batch_size = 16         # 减小批次大小
hidden_size = 256       # 增加隐藏层大小
```

**预期现象**：
- 学习率变小 → 收敛更慢但更稳定
- batch_size变小 → 波动更大，可能泛化更好
- hidden_size变大 → 拟合能力更强，可能过拟合

#### 实验2：替换激活函数
```python
# 原代码：ReLU
def activation(x):
    return np.maximum(0, x)

# 修改为：Leaky ReLU
def activation(x):
    return np.where(x > 0, x, 0.01 * x)

# 修改为：Sigmoid
def activation(x):
    return 1 / (1 + np.exp(-x))
```

#### 实验3：消融实验
```python
# 原代码：有残差连接
def residual_block(x):
    shortcut = x
    x = conv1(x)
    x = activation(x)
    x = conv2(x)
    return x + shortcut  # 残差连接

# 修改为：无残差连接
def residual_block(x):
    # shortcut = x  # 注释掉
    x = conv1(x)
    x = activation(x)
    x = conv2(x)
    return x  # 不要 + shortcut
```

**如何判断改进**：
1. **训练loss**：是否下降更快/更低？
2. **验证loss**：是否泛化更好？
3. **可视化**：注意力图/特征图是否更清晰？
4. **计算效率**：是否更快/更省内存？

### Q16：RAG怎么用到我的项目里？
**一句话结论**：**检索 + 生成**，两步走，先找资料再回答。

**最小RAG架构**：
```
用户问题 → 检索系统 → 相关文档 → 大模型 → 答案
```

**3步实现法**：

#### 第1步：构建检索系统
```python
# 1. 文档处理
documents = ["文档1内容", "文档2内容", ...]

# 2. 向量化（论文28）
encoder = DenseEncoder()
doc_vectors = encoder.encode(documents)

# 3. 建立索引
index = build_index(doc_vectors)
```

#### 第2步：实现检索逻辑
```python
def retrieve(query, top_k=3):
    # 查询向量化
    query_vector = encoder.encode([query])
    
    # 相似度计算
    similarities = cosine_similarity(query_vector, doc_vectors)
    
    # 返回top_k相关文档
    top_indices = np.argsort(similarities)[-top_k:]
    return [documents[i] for i in top_indices]
```

#### 第3步：生成答案
```python
def generate_answer(query, retrieved_docs):
    # 构建prompt
    context = "\n\n".join(retrieved_docs)
    prompt = f"基于以下信息回答问题：\n\n{context}\n\n问题：{query}\n答案："
    
    # 调用大模型（或本地小模型）
    answer = llm.generate(prompt)
    return answer
```

### Q19：怎么评估模型效果？
**一句话结论**：**看多个指标**，不只是准确率。

**常用评估指标**：

#### 分类任务
- **准确率**：正确分类的比例
- **精确率**：预测为正的样本中，真正为正的比例
- **召回率**：真正为正的样本中，被正确预测的比例
- **F1分数**：精确率和召回率的调和平均

#### 生成任务
- **BLEU**：机器翻译评估
- **ROUGE**：文本摘要评估
- **Perplexity**：语言模型困惑度

#### 检索任务
- **召回率@K**：前K个结果中包含正确答案的比例
- **平均精度**：考虑排名的精度
- **NDCG**：考虑相关度的排序质量

**评估流程**：
1. **划分数据集**：训练集（70%）、验证集（15%）、测试集（15%）
2. **训练阶段**：用验证集调参
3. **最终评估**：用测试集（只测一次！）

### Q20：怎么避免过拟合？
**一句话结论**：**正则化 + 早停 + 数据增强**。

**过拟合症状**：
- 训练loss很低，验证loss很高
- 在训练集上完美，在新数据上很差
- 模型记住了训练数据，没学会规律

**解决方案**：

#### 1. 正则化（论文4）
```python
# Dropout
def dropout(x, rate=0.5):
    mask = np.random.binomial(1, 1-rate, size=x.shape)
    return x * mask / (1-rate)

# L2正则化
loss = original_loss + lambda * np.sum(weights**2)
```

#### 2. 早停（Early Stopping）
```python
best_val_loss = float('inf')
patience = 10  # 容忍次数
counter = 0

for epoch in range(100):
    train()
    val_loss = evaluate()
    
    if val_loss < best_val_loss:
        best_val_loss = val_loss
        counter = 0
        save_model()  # 保存最佳模型
    else:
        counter += 1
        if counter >= patience:
            break  # 停止训练
```

#### 3. 数据增强
```python
# 图像增强
augmented_images = []
for img in images:
    # 随机旋转
    rotated = rotate(img, angle=np.random.uniform(-30, 30))
    # 随机裁剪
    cropped = random_crop(rotated)
    # 随机颜色抖动
    jittered = color_jitter(cropped)
    augmented_images.append(jittered)
```
